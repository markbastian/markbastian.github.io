Clojure

Clojure collections are...

  • Java Collections (read-only)
  • Persistent
  • Heterogeneous
  • Nestable
  • Functions of Themselves

Clojure collections *are* Java Collections

  • Makes for excellent interop
  • Caveats
    • Heterogeneity
    • Read-only interfaces
  • To maximize compatibility use into

Clojure collections are Persistent

  • Updates are destructuve
  • Hallmark of FP
  • Implement read-only Java interfaces
  • Can make creation in Java clumsy
  • In Java use Java collections, noting round-tripping is read-only

Clojure collections are Heterogeneous

  • You can put anything in a collection
  • For Java interop you would use a cast
  • Sounds hard, but access and updates are trivial
  • (get-in c [a 1 b 3])
  • (assoc-in c [a 1 b 3] 4)
  • (update-in c [a 1 b 3] inc)

Clojure collections are Nestable

  • You can put anything in a collection
  • ...including collections
  • ...or collections or collections
  • Access and updates are trivial

Clojure collections are Functions of Themselves

  • Think about this for a second
  • y = f(x)
  • A map is a function of its keys
  • An array is a function of its indices
  • A set is a function of its elements
